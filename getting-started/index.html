Get started with Konstruct!

We recommend getting up and running with the Konstruct boilerplate, a separate project that comes with a few extra goodies to help you manage your project a bit better.

Out of the box, Konstruct is just a SASS library, but with the boilerplate you get all of the extra benefits from tools such as:

  • Bower
  • Gulpjs
  • Git Friendly setup
  • Mostly automated workflow
  • Starter HTML & File Structuring.

Sounds great? Lekker, let's get you started!

Before you begin:

The boilerplate assumes that you have Nodejs, Sass, Gulpjs & Bower installed. If you don't, or don't know what these are, please go walk through this article, that should point you in the right direction.

Beware:

The Konstruct boilerplate is still very much in alpha release. While we believe in our new direction, you should always err on the side of caution. It's very important to read the release notes & known bugs section of your boilerplate version before committing it to a production website.

1. Get the boilerplate.

The simplest way to get hold of the boilerplate is to clone the project from Github:

git clone https:// [YOUR_FOLDER]

Once you have cloned the project, simply cd into your working folder, and move on to the next steps!

2. Install Gulpjs.

Next we need to get GulpJS up and running. If you have NodeJS & Gulp installed, this should be as simple as running:

npm install

Once this has successfully run (might take a while), you can move on over to the next step in the process. Almost there!

3. Get Bower Dependencies

There are 2 ways to do this, the default method of running bower install, or the recommended method through GulpJS. Running through Gulp will give you a much more reliable setup, and should get you building your awesome site in minutes!

Step 3.1 - Install Bower Dependencies

gulp bower

Step 3.2 - Install Konstruct to the correct folders

gulp install-konstruct

Step 3.3 - Clean up errant folders, and get rid of .git folder.

gulp clean

4. Test!

Let's just make sure everything wen't okay shall we? If you run gulp test, it will run a few test tasks to make sure everything is where it's expected to be. If you get the 'All Clear!', you can move on to the final step - building your site!

5. Go Build Something Awesome!

Very Important:

We are not currently sure what will happen if you run the install a second time. We expect everything to be overridden. Please make backups while you help us test.

You are ready to start building! Below is a quick menu of gulp commands you can use during each chapter of your workflow. If you forget them, you can also just run gulp and have them repeated for you.

Gulp Commands

  • 1. gulp - Will show you the available commands at your disposal.
  • 2. gulp develop - Will watch as you work, compile your sass as you save, add vendor prefixes and livereload the page.
  • 3. gulp build - Coming soon.
  • 4. gulp deploy - Coming soon.
